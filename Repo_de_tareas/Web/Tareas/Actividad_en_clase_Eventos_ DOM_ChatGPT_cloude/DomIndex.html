Preguntas

  1. ¿Qué es el DOM?

    2. El DOM (Document Object Model) es una representación en memoria de la estructura de un documento HTML/XML que permite a los programas acceder y manipular el contenido, estructura y estilo del documento. El DOM representa el documento como un árbol de nodos, donde cada parte del documento (elementos, atributos, texto) es un nodo.

    // Ejemplo de manipulación del DOM document.getElementById("miElemento").innerHTML = "Nuevo contenido";
  2. ¿Qué es un nodo?

    3. Un nodo es la unidad básica en el DOM. Cada parte del documento HTML es representada como un nodo. Existen diferentes tipos de nodos:

    • Element Node: Representa etiquetas HTML (div, p, h1, etc.)
    • Text Node: Representa el texto dentro de un elemento
    • Attribute Node: Representa atributos de elementos HTML
    • Comment Node: Representa comentarios en el HTML
    • Document Node: Representa el documento completo
    // Ejemplo de creación de un nodo const parrafo = document.createElement("p"); // Crea un nodo de elemento const texto = document.createTextNode("Hola mundo"); // Crea un nodo de texto parrafo.appendChild(texto); // Añade el nodo de texto al nodo de elemento
  3. ¿Cómo se accede a un elemento del DOM?

    4. Existen varios métodos para acceder a elementos del DOM:

    • getElementById(): Selecciona un elemento por su atributo ID
    • getElementsByClassName(): Selecciona elementos por su clase
    • getElementsByTagName(): Selecciona elementos por su etiqueta
    • querySelector(): Selecciona el primer elemento que coincida con un selector CSS
    • querySelectorAll(): Selecciona todos los elementos que coincidan con un selector CSS
    // Ejemplos de acceso a elementos del DOM const elementoPorId = document.getElementById("miId"); const elementosPorClase = document.getElementsByClassName("miClase"); const parrafos = document.getElementsByTagName("p"); const primerBoton = document.querySelector("button"); const todosLosInputs = document.querySelectorAll("input[type='text']");
  4. ¿Cómo se modifica un elemento del DOM?

    5. Existen varias formas de modificar elementos del DOM:

    • Modificar contenido: innerHTML, textContent, innerText
    • Modificar atributos: setAttribute(), getAttribute(), removeAttribute()
    • Modificar estilos: style.propiedad, classList.add(), classList.remove(), classList.toggle()
    • Crear y eliminar elementos: createElement(), appendChild(), removeChild(), replaceChild()
    // Ejemplos de modificación de elementos document.getElementById("miElemento").innerHTML = "Nuevo contenido"; document.getElementById("miImagen").setAttribute("src", "nueva-imagen.jpg"); document.getElementById("miDiv").style.backgroundColor = "blue"; document.getElementById("miDiv").classList.add("destacado"); // Crear y añadir un nuevo elemento const nuevoElemento = document.createElement("p"); nuevoElemento.textContent = "Texto nuevo"; document.body.appendChild(nuevoElemento);
  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    6. Las principales diferencias entre los métodos getElementBy* y querySelector* son:

    getElementBy* querySelector*
    Devuelve HTMLCollection (live collection) Devuelve NodeList (static collection)
    Métodos específicos para cada tipo de selección (ID, clase, etiqueta) Un método general que acepta cualquier selector CSS
    Mejor rendimiento en selecciones simples Más versátil para selecciones complejas
    No permite combinaciones de selectores Permite usar toda la potencia de los selectores CSS
    // getElementBy* ejemplo const elementoPorId = document.getElementById("miId"); // Un solo elemento const elementosPorClase = document.getElementsByClassName("miClase"); // HTMLCollection // querySelector* ejemplo const primerElemento = document.querySelector("#miId .miClase"); // Primer elemento que coincida const todosLosElementos = document.querySelectorAll("div > p"); // NodeList con todos los elementos
  6. ¿Qué es un evento?

    7. Un evento es una señal de que algo ha ocurrido en la página web, como interacciones del usuario (clicks, movimientos del ratón, pulsaciones de teclas) o cambios en el entorno del navegador (carga de la página, redimensionamiento de la ventana). Los eventos permiten que JavaScript reaccione a estas acciones.

    Ejemplos comunes de eventos:

    • click: Se activa cuando el usuario hace clic en un elemento
    • mouseover/mouseout: Se activan cuando el puntero entra o sale de un elemento
    • keydown/keyup: Se activan cuando se presiona o suelta una tecla
    • submit: Se activa cuando se envía un formulario
    • load: Se activa cuando el documento o un recurso ha terminado de cargarse
    // Ejemplo de evento click document.getElementById("miBoton").addEventListener("click", function() { alert("¡Botón clickeado!"); });
  7. ¿Qué es un listener?

    8. Un listener (o escuchador de eventos) es una función que "escucha" y responde cuando ocurre un evento específico. Los listeners se asocian a elementos del DOM mediante el método addEventListener(), que especifica qué tipo de evento escuchar y qué hacer cuando ocurra.

    // Sintaxis de addEventListener elemento.addEventListener(tipoEvento, funcionManejadora, opciones); // Ejemplo práctico const boton = document.getElementById("miBoton"); function manejarClick() { console.log("El botón fue clickeado"); } // Añadir un listener para el evento "click" boton.addEventListener("click", manejarClick); // Eliminar un listener boton.removeEventListener("click", manejarClick);
  8. ¿Qué es un callback?

    9. Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que ocurra algún evento o se complete alguna operación. En el contexto del DOM, los callbacks son comúnmente utilizados como manejadores de eventos (event handlers).

    Los callbacks son fundamentales en JavaScript para manejar operaciones asíncronas y eventos.

    // Ejemplo de callback en un evento document.getElementById("miBoton").addEventListener("click", function() { // Esta función anónima es un callback console.log("Botón clickeado"); }); // Ejemplo de callback con función nombrada function manejarClick() { console.log("Botón clickeado"); } document.getElementById("miBoton").addEventListener("click", manejarClick); // Ejemplo de callback en una operación asíncrona setTimeout(function() { console.log("Este mensaje aparece después de 2 segundos"); }, 2000);
  9. ¿Qué es un objeto de evento?

    10. Un objeto de evento es un objeto que se crea automáticamente cuando ocurre un evento y se pasa como parámetro a la función de callback del evento. Este objeto contiene información detallada sobre el evento ocurrido, como:

    • type: El tipo de evento (click, keydown, etc.)
    • target: El elemento que desencadenó el evento
    • clientX/clientY: Las coordenadas del puntero en eventos de ratón
    • key: La tecla presionada en eventos de teclado
    • preventDefault(): Método para prevenir la acción predeterminada del evento
    • stopPropagation(): Método para detener la propagación del evento
    // Ejemplo de uso del objeto de evento document.getElementById("miInput").addEventListener("keydown", function(evento) { console.log("Tecla presionada: " + evento.key); console.log("Código de la tecla: " + evento.keyCode); // Prevenir la acción predeterminada (escribir en el input) if (evento.key === "Enter") { evento.preventDefault(); console.log("Se presionó Enter"); } }); // Ejemplo con evento de ratón document.getElementById("miDiv").addEventListener("mousemove", function(evento) { console.log("Posición X: " + evento.clientX); console.log("Posición Y: " + evento.clientY); console.log("Elemento objetivo: " + evento.target.id); });

Conclusiones

Experiencia general

Desesperante, frustrante, al depende al 100% de la IA, no te dará una solución a la primera, se necesitan varios intentos para que claude te de la respuesta correcta

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Solo lo usaria para programas que no encuentre el error y solo usarlo para referencias, otro sería usarlo para conocer o recordar funciones que uno no se acuerda

Ventajas de usar AI para programar

  • Es una gran ayuda por si estás atorado con un programa
  • La información es más precisa a comparación de hacer una busqueda de internet
  • Puedes aprender mucho si usas la IA de un lado educativo

Desventajas de usar AI para programar

  • Puedes desarrollar dependencia a la IA, lo que puede ser perjudicial a la hora de hacer un trabajo independiente
  • La IA comete muchos errores, lo que puede frustrar
  • La IA no sabe lo que quieres, debes aclarar varias veces hasta que cree la respuesta
  • Muchas IAs son de pago, por ende, si quieres usar una más inteligente, no puedes, tienes que gastar

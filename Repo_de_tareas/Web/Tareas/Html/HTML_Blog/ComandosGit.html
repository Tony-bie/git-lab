Comandos de git/github

Proceso para subir un archivo y sus funciones

  1. Git add

    2. Este comando lo que hace es darle seguimiento los cambios hechos en el repositorio

    Variantes:

    Git add {name} Git add .
    Sigue los cambios hechos de un
    archivo específico     		Sigue los cambios hechos
    de todos los archivos

  2. Git commit -m {Comment}

    3. Este comando tras usar el git add, comenta los cambios hecho de un repositorio, muy útil para aclarar que cambios se hicieron.

  3. Git push

    4. Este comando es el proceso final, es para guardar los cambios hechos y subirlo tanto a tu repositorio local como en línea.

Como crear branches y sus funciones

Un branch permite trabajar en diferentes versiones sin afectar la versión original,
con tal de no hacer cambios que puedan afectar a la versión original

  1. Git branch

    2. Te muestra todos los branches creados

  2. Git branch {nombre}

    3. Crea un branch con el nombre que pusiste

  3. Git switch {name}

    4. Te permite moverte entre cada branch

  4. Git push -u origin {name}

    5. Te permite guardar los cambios y subirlo a los repositorios dentro de un branch

Otras funciones

  • Git pull

    • Lo que hace es tomar los cambios hechos en el repositorio de github al repositorio local

  • Git status

    • Te dice que archivos tienen seguimientos y cuales no

  • Git log

    • Con esto puedes ver el historial de comandos usados durante la vida del repositorio
